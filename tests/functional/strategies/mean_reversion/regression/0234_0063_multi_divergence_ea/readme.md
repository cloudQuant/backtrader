# 0063 Multi Divergence EA

## 策略来源

- MT5 源码：`ea/0063_Multi_Divergence_EA/multi_divergence.mq5`

## 策略逻辑

- 同时扫描 `RSI`、`MACD` 主线和 `Stochastic` 主线的价格背离。
- 若价格创新低而指标低点抬高，则记为看涨背离；若价格创新高而指标高点降低，则记为看跌背离。
- 只有当至少 `MinConfirmations` 个指标同时给出同向背离时，才触发交易信号。
- 可选 `EMA50` 趋势过滤：只在价格高于 `EMA50` 时做多、低于 `EMA50` 时做空。
- 可选成交量过滤：最近一根成交量需要高于过去 `10` 根平均成交量的 `1.2` 倍。
- 入场后使用固定 `StopLoss / TakeProfit`。
- 若启用资金管理，则按照 `RiskPercent` 和固定止损距离估算下单手数；否则使用固定 `LotSize`。

## 与源码一致/差异说明

- 保留了三类振荡指标的背离扫描、多指标确认、EMA 趋势过滤、成交量过滤、点差过滤和固定 `SL/TP` 主流程。
- 原源码在 `PERIOD_CURRENT` 上运行，当前迁移版本使用 `XAUUSD_M15.csv` 重采样到 `H1` 进行回测，以贴近原说明中推荐的 `H1/H4/D1` 较高周期使用场景。
- `MaxTrades` 在 MT5 中代表同品种同 magic 的最大并发仓位数；当前 backtrader 迁移版本按单一聚合持仓处理，默认参数 `1` 可忠实对应。
- 原源码的成交量使用 tick volume；当前版本使用 CSV 中的 `<TICKVOL>` 字段并在重采样后求和，属于合理近似。

## 当前验证结果

- 使用默认参数直接运行 `python run.py` 时，策略可正常完成回测，但在当前 `XAUUSD_M15 -> H1` 数据样本上未触发交易。
- 进一步诊断表明，放宽到更宽松的确认条件后，迁移版本可以正常产生并执行交易，因此当前默认结果更像是参数与样本窗口过严，而不是实现失效。
- 因此本示例当前状态可视为：**迁移代码已跑通，默认参数在现有样本上无成交**。

## 运行方式

```bash
python run.py
```

- `config.yaml`：策略参数、数据区间和回测配置。
