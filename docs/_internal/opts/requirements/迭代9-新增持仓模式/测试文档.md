## 1. 测试策略

### 1.1 总体原则

1. **先保净仓，再上双向**：`net` 模式零退化是第一优先级
2. **按能力分层**：参数与签名 -> 订单元信息 -> 持仓腿执行 -> 现金/PnL -> 通知
3. **兼容回归必须显式测试**：尤其是 `BtApiBroker.getposition(data, clone=True)`、`Strategy.position`、`broker.positions`
4. **不测试文档未承诺的能力**：P0 不把 bracket 包装函数纳入通过标准

### 1.2 建议目录

| 层级 | 建议路径 |
|------|---------|
| 单元 | `tests/unit/brokers/test_dual_side_params.py` |
| 单元 | `tests/unit/brokers/test_dual_side_order_metadata.py` |
| 单元 | `tests/unit/brokers/test_dual_side_backbroker.py` |
| 单元 | `tests/unit/brokers/test_dual_side_tickbroker.py` |
| 单元 | `tests/unit/brokers/test_dual_side_btapibroker.py` |
| 单元 | `tests/unit/core/test_strategy_dual_side.py` |

说明：

1. 不再使用仓库中不存在的 `tests/add_tests/...` 路径
2. 双向持仓测试尽量贴近现有 `tests/unit/brokers` 和 `tests/integration` 组织方式

## 3. 特殊兼容回归点

这是原始测试文档里遗漏、但本轮必须补齐的回归点。

### 3.1 `BtApiBroker.getposition(..., clone=True)` 兼容

必须覆盖：

```python
position = broker.getposition(data)
cached = broker.getposition(data, clone=False)
long_pos = broker.getposition(data, clone=False, side="long")
```

预期：

1. 默认仍返回 clone
2. `clone=False` 仍返回缓存对象
3. 追加 `side` 后不改变 `clone` 行为

### 3.2 `broker.positions` / `Strategy.position`

必须覆盖：

1. `broker.positions[data]` 在 dual-side 下仍可读取净头寸兼容视图
2. `Strategy.position` 仍可用
3. 当长短腿同时存在时，`position.price` 仅视为兼容字段，不作为双向结算依据

### 3.3 TickBroker 未知 kwargs 路径

必须覆盖：

1. `TickBroker.buy(..., position_side="long", offset="open")` 不会因为 `OrderParams` 未声明字段而抛 `AttributeError`
2. `MixBroker` 自动继承相同行为

### 4.2 `BtApiBroker` 兼容签名测试

```python
def test_btapibroker_getposition_keeps_clone_compatibility(data, store):
    broker = BtApiBroker(store=store, position_mode="dual_side")
    broker.positions["TEST"].fix(0, 0.0)
    broker.long_positions["TEST"].fix(2, 100.0)
    broker.short_positions["TEST"].fix(1, 105.0)
    broker._sync_net_position("TEST")

    pos1 = broker.getposition(data)
    pos2 = broker.getposition(data, clone=False)
    pos3 = broker.getposition(data, clone=False, side="long")

    assert pos1 is not pos2
    assert pos2.size == 1
    assert pos3.size == 2
```

### 4.3 `close()` 歧义测试

```python
def test_close_raises_when_both_legs_exist(strategy):
    strategy.buy(size=100, position_side="long", offset="open")
    strategy.sell(size=50, position_side="short", offset="open")

    with pytest.raises(ValueError, match="position_side"):
        strategy.close()
```

## 5. 性能测试建议

### 5.1 基准项

| 场景 | 目标 |
|------|------|
| `net` 改造前后对比 | 退化 < 2% |
| dual-side 基础回测 | 建立基线 |
| TickBroker dual-side | 无明显超线性退化 |

### 5.2 重点观察

1. `net` 模式是否因为双向分支判断造成普遍性能回退
2. `positions` 净视图同步是否引入热点开销
3. `TickBroker._execute()` 与 `order_history` 是否因附加字段导致显著放大

## 7. 风险提示

| 风险 | 说明 | 缓解 |
|------|------|------|
| 只测开平仓，不测兼容签名 | 很容易破坏 `BtApiBroker clone` 行为 | 单独增加签名回归 |
| 只测 broker，不测 Strategy | `close()` / `getposition()` 透传问题会漏掉 | 增加 `tests/unit/core/test_strategy_dual_side.py` |
| 只测净仓结果，不测 `order.info` | 订单通知无法被策略正确消费 | 增加元信息测试 |
| 只测 bar broker，不测 TickBroker 下单路径 | `OrderParams` 未知 kwargs 问题会漏掉 | 增加 Tick/Mix 单测 |
