## 1. 测试策略

### 1.1 总体原则

| 原则 | 说明 |
|------|------|
| 能力优先 | 测试目标是验证中频回测的新能力，而不是机械沿用旧测试文件结构 |
| 时间语义优先 | 先验证数据可见性和时间对齐，再验证策略收益结果 |
| 分层验证 | 采用规则测试、场景测试、集成测试、样例测试四层结构 |
| 单品种优先 | 先证明单品种主链路成立，再验证多品种联动 |
| 高低频分责 | bar 过滤和高频执行分别验证，避免语义混淆 |
| 可解释性 | 每个关键失败都要能定位是时间问题、上下文问题还是执行问题 |

### 1.2 测试分层

```text
Layer 4: 端到端 / 样例测试
  - 单品种中频样例
  - 多品种联动 / 套利样例
  - 回测结果可解释性验证

Layer 3: 集成测试
  - MixBroker + 多源数据输入
  - 高频优先执行（bar 不参与撮合）
  - 多品种全局时间调度

Layer 2: 场景测试
  - 时间对齐规则
  - 统一快照生成
  - 策略上下文读取
  - 账户回写一致性

Layer 1: 规则测试
  - bar 完成语义
  - 同时刻事件优先级
  - 数据可见性边界
  - bar 仅更新低频状态边界
```

### 1.3 测试口径

1. 本轮以“中频回测行为契约”作为测试主线。
2. 测试不应依赖 `MixBroker` 最终采用继承还是组合，只验证行为结果。
3. 测试重点不在“收益数字”，而在：
   - 数据是否按正确时点可见
   - 策略是否基于正确快照决策
   - 订单是否走了正确执行路径
   - 账户是否被正确回写
4. 多品种测试重点不在复杂收益模型，而在统一时间调度和快照一致性。

## 3. 推荐测试场景

### 3.1 单品种业务样例场景

#### 场景 S-01：高频买盘占优 + 均线向上，触发做多

**输入条件**：

- 过去30个 `orderbook` 样本可计算前10档买卖盘金额比值。
- 比值大于 `1.5`。
- 最近已完成的 1 分钟bar收盘价高于20周期均线。
- 高频执行数据可用。

**预期结果**：

- 策略触发做多。
- 订单优先进入高频执行链路。
- 成交后持仓和现金更新正确。

#### 场景 S-02：高频卖盘占优 + 均线向下，触发做空

**输入条件**：

- 买卖盘金额比值小于 `0.7`。
- 最近已完成bar收盘价低于20周期均线。

**预期结果**：

- 策略触发做空。
- 执行路径仍优先走高频数据。

#### 场景 S-03：高频信号满足，但均线过滤不满足

**预期结果**：

- 不触发交易。
- 能明确区分是过滤条件拦截，而不是数据未对齐。

### 3.2 时间语义边界场景

#### 场景 S-04：bar 尚未完成时不得使用其收盘价

**预期结果**：

- 策略不会提前读取该bar收盘信息。
- 同一时点重放结果一致。

#### 场景 S-05：同一时间戳存在高频事件和bar完成事件

**预期结果**：

- 按既定优先级形成一致快照。
- 决策不会因内部顺序随机变化。

### 3.3 bar 不参与撮合场景

#### 场景 S-06：bar 到达时有未成交订单

**预期结果**：

- 订单只走高频执行路径（tick/orderbook 撮合）。
- bar 仅更新低频状态，不参与成交。

#### 场景 S-07：仅有 bar 数据无高频数据

**预期结果**：

- 订单始终不会被 bar 撮合。
- 策略可正常读取 bar 衍生指标（均线等）。
- 订单保持未成交，直到后续高频事件到达或被策略撤销。

### 3.4 多品种联动场景

#### 场景 S-08：A 合约状态触发 B 合约下单

**预期结果**：

- 策略可以读取 A 与 B 在当前全局时点的可见状态。
- B 合约订单按自己的高频路径执行。

#### 场景 S-09：双合约套利

**预期结果**：

- 两个合约可同时生成信号和订单。
- 组合账户持仓与现金汇总正确。

## 6. 已知测试风险

| 风险 | 缓解 |
|------|------|
| 时间戳边界导致偶发行为差异 | 增加相同时间戳与边界时间场景测试 |

## 7. 验收结论口径

本轮测试最终要回答的不是"是否赚钱"，而是以下问题：

1. 中频回测的时间语义是否清晰且稳定。
2. 策略是否基于正确的统一快照做决策。
3. 执行是否坚持高频优先语义。
4. 多品种场景下是否具备一致的全局调度和账户回写逻辑。
5. 文档、设计、任务和测试口径是否一致。

#### 8.1.1 最小单品种测试数据集

用于 Phase 1-3 的基础测试，覆盖 tick / orderbook / bar 三类事件的混合输入：

```python
# tests/datas/midfreq/build_test_data.py

def build_single_symbol_test_data():
    """构造单品种中频回测最小测试数据集。

    输出：
    - 100 个 tick 事件（每秒 1 个，10:00:00 - 10:01:39）
    - 50 个 orderbook 事件（每 2 秒 1 个，与 tick 交错）
    - 2 个 bar 事件（10:01:00 和 10:02:00 各一个 1 分钟 K 线）

    关键设计：
    - bar 的 timestamp = 结束时间，如第一个 bar 的 ts = 10:01:00
    - 在 10:01:00 时刻同时存在 tick 和 bar，用于验证同时刻排序
    - orderbook 的 bids/asks 各 10 档，价格围绕 100.0 波动
    """
    ticks = []
    orderbooks = []
    bars = []

    base_ts = 1700000000.0  # 基准时间戳

    # 生成 tick 事件
    for i in range(100):
        ts = base_ts + i  # 每秒 1 个
        ticks.append({
            "channel_type": "tick",
            "symbol": "BTCUSDT",
            "timestamp": ts,
            "price": 100.0 + (i % 10) * 0.1,
            "volume": 1.0,
        })

    # 生成 orderbook 事件
    for i in range(50):
        ts = base_ts + i * 2 + 0.5  # 每 2 秒 1 个，偏移 0.5s
        bid_base = 99.5 + (i % 5) * 0.1
        ask_base = 100.5 + (i % 5) * 0.1
        orderbooks.append({
            "channel_type": "orderbook",
            "symbol": "BTCUSDT",
            "timestamp": ts,
            "bids": [(bid_base - j * 0.1, 10.0 + j) for j in range(10)],
            "asks": [(ask_base + j * 0.1, 10.0 + j) for j in range(10)],
        })

    # 生成 bar 事件（bar 的 timestamp = 结束时间）
    bars.append({
        "channel_type": "bar",
        "symbol": "BTCUSDT",
        "timestamp": base_ts + 60,  # 10:01:00
        "timeframe": "1min",
        "open": 100.0, "high": 100.9, "low": 99.8, "close": 100.5,
        "volume": 500.0,
    })
    bars.append({
        "channel_type": "bar",
        "symbol": "BTCUSDT",
        "timestamp": base_ts + 120,  # 10:02:00
        "timeframe": "1min",
        "open": 100.5, "high": 101.2, "low": 100.1, "close": 100.8,
        "volume": 600.0,
    })

    return ticks, orderbooks, bars
```

class TestMixedChannel:

    def test_merged_timestamps_non_decreasing(self):
        """归并后事件流时间戳严格非递减。"""
        ticks, obs, bars = build_single_symbol_test_data()
        merged = build_mixed_channel(ticks, obs, bars)
        timestamps = [e.data.timestamp for e in merged]
        for i in range(1, len(timestamps)):
            assert timestamps[i] >= timestamps[i - 1]

    def test_same_timestamp_order(self):
        """同时刻事件排序：tick -> orderbook -> bar。"""
        events = build_same_timestamp_test_data()
        merged = build_mixed_channel_from_events(events)
        types = [e.channel_type for e in merged]
        assert types == ["tick", "orderbook", "bar"]

    def test_multi_symbol_global_order(self):
        """多品种事件按全局时间戳统一排列。"""
        data_a, data_b = build_multi_symbol_test_data()
        merged = build_mixed_channel(data_a, data_b)
        timestamps = [e.data.timestamp for e in merged]
        for i in range(1, len(timestamps)):

    def test_event_count_preserved(self):
        """归并后事件总数 = 输入事件总数（不丢失不重复）。"""
        ticks, obs, bars = build_single_symbol_test_data()
        merged = list(build_mixed_channel(ticks, obs, bars))

# tests/test_midfreq_context.py

class TestMidFreqContext:

    def test_ob_ratio_calculation(self):
        """验证 OB 比值计算正确性。"""
        broker = MixBroker(cash=100000)
        broker.start()
        # 注入 30 个 orderbook，bids 金额 > asks 金额
        for i in range(30):
            ob = make_ob_event(
                symbol="BTCUSDT",
                bids=[(100 - j * 0.1, 20) for j in range(10)],
                asks=[(101 + j * 0.1, 10) for j in range(10)],
            )
            broker.process_orderbook(ob)
        ctx = broker.get_context()
        ratio = ctx.get_ob_ratio("BTCUSDT", levels=10, window=30)
        assert ratio is not None
        assert ratio > 1.0  # bids 金额 > asks 金额

    def test_sma_correctness(self):
        """验证 SMA 均线计算正确性。"""
        broker = MixBroker(cash=100000)
        broker.start()
        closes = [100 + i * 0.5 for i in range(20)]
        for i, close in enumerate(closes):
            bar = make_bar_event(
                symbol="BTCUSDT",
                close=close, open=close - 0.1,
                high=close + 0.2, low=close - 0.2,
                timestamp=1700000000 + (i + 1) * 60,
            )
            broker.process_bar(bar)
        ctx = broker.get_context()
        sma = ctx.get_sma("BTCUSDT", period=20)
        expected = sum(closes) / 20
        assert abs(sma - expected) < 1e-10

    def test_context_readonly(self):
        """验证 context 不暴露可变引用。"""
        broker = MixBroker(cash=100000)
        broker.start()
        ctx = broker.get_context()
        window = ctx.get_ob_window("BTCUSDT", 10)
        original_len = len(window)
        window.append("garbage")
        assert len(ctx.get_ob_window("BTCUSDT", 10)) == original_len

    def test_no_data_returns_none(self):
        """无数据时查询返回 None。"""
        broker = MixBroker(cash=100000)
        broker.start()
        ctx = broker.get_context()
        assert ctx.get_ob_ratio("BTCUSDT") is None
        assert ctx.get_sma("BTCUSDT") is None
        assert ctx.get_last_tick("BTCUSDT") is None

# tests/bench/bench_midfreq.py

import time

class TestMidFreqPerformance:

    def test_single_symbol_throughput(self):
        """单品种中频回测性能基准。

        目标：10 万 tick + 1000 bar 运行时间 < 30 秒。
        """
        broker = MixBroker(cash=100000, max_ob_window=100)
        broker.start()

        start = time.perf_counter()
        for i in range(100_000):
            tick = make_tick_event(
                symbol="BTCUSDT", price=100 + i * 0.001, timestamp=i * 0.1
            )
            broker.process_tick(tick)
            if i % 100 == 0:
                bar = make_bar_event(
                    symbol="BTCUSDT", close=100 + i * 0.001, timestamp=i * 0.1
                )
                broker.process_bar(bar)
        elapsed = time.perf_counter() - start
        assert elapsed < 30.0, f"耗时 {elapsed:.2f}s 超过 30s 基线"

    def test_ob_ratio_latency(self):
        """get_ob_ratio() 单次调用耗时 < 1ms。"""
        broker = MixBroker(cash=100000, max_ob_window=100)
        broker.start()
        for i in range(100):
            ob = make_ob_event(
                symbol="BTCUSDT",
                bids=[(100 - j * 0.1, 10) for j in range(10)],
                asks=[(101 + j * 0.1, 10) for j in range(10)],
            )
            broker.process_orderbook(ob)
        ctx = broker.get_context()

        start = time.perf_counter()
        for _ in range(1000):
            ctx.get_ob_ratio("BTCUSDT", levels=10, window=30)
        avg_ms = (time.perf_counter() - start) / 1000 * 1000
        assert avg_ms < 1.0, f"平均耗时 {avg_ms:.3f}ms 超过 1ms 基线"

    def test_memory_overhead(self):
        """单品种内存增量 < 5 MB。"""
        import tracemalloc
        tracemalloc.start()
        broker = MixBroker(
            cash=100000,
            max_ob_window=100, max_bar_history=200,
        )
        broker.start()
        snapshot_before = tracemalloc.take_snapshot()

        for i in range(100):
            ob = make_ob_event(
                symbol="BTCUSDT",
                bids=[(100 - j * 0.1, 10) for j in range(20)],
                asks=[(101 + j * 0.1, 10) for j in range(20)],
            )
            broker.process_orderbook(ob)
        for i in range(200):
            bar = make_bar_event(
                symbol="BTCUSDT", close=100 + i * 0.1, timestamp=i * 60
            )
            broker.process_bar(bar)

        snapshot_after = tracemalloc.take_snapshot()
        stats = snapshot_after.compare_to(snapshot_before, "lineno")
        total_diff = sum(s.size_diff for s in stats if s.size_diff > 0)
        assert total_diff < 5 * 1024 * 1024, (
            f"内存增量 {total_diff / 1024 / 1024:.2f} MB 超过 5 MB"
        )
        tracemalloc.stop()
