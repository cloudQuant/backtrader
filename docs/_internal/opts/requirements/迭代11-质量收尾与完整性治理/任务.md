#### P0.1 策略测试存在语法错误文件 🔴

  3. 验收：`python -c "import ast; ast.parse(...)"` 通过 + `pytest <file>` 通过 +
     `git diff` 仅注释/docstring 变更。
- **验收命令**：
  ```bash

    tests/functional/strategies/mean_reversion/test_0138_0600_bollinger_n_positions.py
  pytest tests/functional/strategies/mean_reversion/test_0138_0600_bollinger_n_positions.py -q
  ```

- **重建台账机制**（迭代10 台账已丢失，必须重建且纳入 git 防再丢）：
  - 新台账文件：本目录下 `领取台账.md`（与本计划同目录），**必须 `git add`

  - 每个文件领取→`进行中`→验收通过后改 `验收通过`，记录验收命令与改动摘要。
- **单文件标准流程**（与迭代10 保持一致）：
  1. `python scripts/analyze_docstrings.py <file>` 确认存在缺口。
  2. `pytest <file>` 修改前必须先通过（不通过则登记为「阻塞」并跳过）。
  3. 补 module header 三章节（`Data Used:` / `Strategy Principle:` /
     `Strategy Logic:`）+ 逐项补类/函数/方法 docstring；中文注释翻译为英文。
  4. 验收：`analyze_docstrings.py` 显示 `0 / 0` + `pytest` 通过 +
     **AST 去 docstring 后与 HEAD 一致**（证明零逻辑改动）。
- **AST 一致性校验脚本**（建议固化为复用工具，防止「只加注释」被违反）：
  ```bash
  python - <<'PY'
  import ast, subprocess, sys
  f = sys.argv[1]
  head = subprocess.check_output(["git","show",f"HEAD:{f}"]).decode()
  cur = open(f).read()
  def strip(src):
      t = ast.parse(src)
      for n in ast.walk(t):
          if isinstance(n,(ast.Module,ast.FunctionDef,ast.AsyncFunctionDef,ast.ClassDef)):
              b=n.body
              if b and isinstance(b[0],ast.Expr) and isinstance(getattr(b[0],'value',None),ast.Constant) and isinstance(b[0].value.value,str):
                  n.body=b[1:] if len(b)>1 else [ast.Pass()]
      return ast.dump(ast.parse(ast.unparse(t)))
  print("MATCH" if strip(head)==strip(cur) else "MISMATCH")
  PY
  ```
- **重要风险提示**：迭代10 期间发现过历史会话把 `test_*` 主体逻辑（如
  `_invoke_strategy_main` 钩子）误改写成简化版的情况。本迭代验收**必须**跑
  上述 AST 校验，发现 `MISMATCH` 一律回退逻辑改动只保留注释。
