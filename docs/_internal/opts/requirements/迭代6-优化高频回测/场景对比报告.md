# 高频场景对比报告

## 1. 目标

本报告用于对比以下三类从 `hftbacktest` 迁移到 `backtrader` 的高频场景在当前实现下的行为一致性：

- `High-Frequency Grid Trading.ipynb`
- `Queue-Based Market Making in Large Tick Size Assets.ipynb`
- `Market Making with Alpha - Order Book Imbalance.ipynb`
- 真实 `tick CSV + orderbook JSONL` 文件回放集成场景

对比口径统一为：

- `reference`：轻量参考 runner
- `backtrader`：`TickBroker + QueueExchangeModel`

对比维度：

- `cash`
- `position`
- `fills`
- `trade_count`
- `state_values`

  - `TickChannel`
  - `OrderBookChannel`
  - `StreamingEventQueue`
  - `Cerebro`
  - `TickBroker`
  - strategy `notify_tick / notify_orderbook`
- replay scope:
  - tick CSV 前 `250` 行
  - orderbook JSONL 前 `120` 行
- assertions:
  - tick 事件数大于 `0`
  - orderbook 事件数大于 `0`
  - 至少完成 `2` 笔成交
  - 第一笔为 `buy`
  - 最后一笔为 `sell`
  - `broker.tick_count > 0`
  - `order_history` 已产生记录
  - `state_values.num_trades >= 2`
  - 最终仓位回到 `0`
  - 最终现金不等于初始现金
- result:
  - `tests/integration/test_hft_csv_orderbook_replay.py` 通过
  - 证明当前实现不仅能处理合成事件，也能处理仓库内真实文件格式的双通道回放

## 4. 结论

当前 `backtrader` 高频实现已经在以下层面与参考 runner 对齐：

- 高频网格挂单
- 队列驱动做市
- 基于订单簿失衡的 alpha 做市
- GTX maker 挂单与成交
- queue ahead 消耗后的 fill 路径
- fill 后现金、仓位与聚合状态统计
- 真实 `tick CSV + orderbook JSONL` 双通道文件回放链路

在本报告覆盖的三类场景中，`reference` 与 `backtrader` 的：

- `cash`
- `position`
- `fills`
- `trade_count`

均完全一致。

## 5. 当前边界

本报告已经证明“核心高频行为实现可用且对齐”，但还没有覆盖：

- 真实历史大样本 orderbook 数据回放
- 更复杂的概率队列模型差异
- maker rebate / taker fee 的细粒度市场规则
- 更长窗口 alpha 策略和更多 inventory 约束
- 与 `hftbacktest` 原生 stats 全指标逐项对照

已经覆盖了“真实文件格式回放”这一层，但当前真实回放仍然是**前缀样本集成验证**，还不是整份大样本历史数据的长期绩效评估。

因此当前结论是：

- **功能层面：已实现且可验证**
- **复杂真实市场微结构层面：仍可继续扩展验证**
