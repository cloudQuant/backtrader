## 1. 关键发现（写清楚要优化的点）

### 发现 A 🔴 — 79 个策略测试文件“永远无法”达到 `Chinese comments: 0`

`scripts/analyze_docstrings.py` 把**任何含中文字符的行**都计入 Chinese comments，
包括配置字典里的 `source_ea` / `source_spec` 字符串**值**：

```python
'source_ea': 'ea/1158_发散交易',
'source_ea': 'ea/1344_MQL5_向导_-_基于_3_乌鸦_3_白兵_+_MFI_...',
'source_spec': '.../策略说明.md',
```

实测：`tests/functional/strategies` 下 **104** 个文件含中文，其中

- **79 个**：中文**只出现在** `source_ea` / `source_spec` 配置值里；
- **25 个**：含真正可翻译的中文注释/docstring。

迭代10 的强制约束写明“不允许修改数据路径、参数、期望值”，而 `source_ea` 正是
**数据来源路径值**。因此这 79 个文件在现有工具口径下**与验收标准（Chinese
comments = 0）直接冲突**：要么改值（违反约束），要么永远验收不过。

**要优化的点**：

1. 给 `analyze_docstrings.py` 的中文检测增加豁免：当中文仅出现在
   `source_ea` / `source_spec`（及其它已知“源路径值”键）所在行时，不计入
   Chinese comments，或单列为 `untranslatable_config_value` 一类。
2. 迭代10/11 验收口径同步更新：`Chinese comments` 统计应排除配置值；新增
   “可翻译中文 = 0”作为真正的验收门槛。
3. 列出这 79 个文件清单，标注为“CJK 仅在源路径值，豁免”，避免被反复误领。

### 发现 B 🟠 — 注释工作量巨大且需全程串行，必须有防冲突机制

`tests/functional/strategies` 仍有 ~658 个文件需补 docstring（~9,461 处），
`tests/` 全量 ~769 个。迭代10 已要求“每次只领一个文件、登记台账、串行验收”，但实践
中出现**并发模型抢同一文件**导致的编辑竞态（同一文件被多方同时改）。

**要优化的点**：

1. 领取台账机制需强约束：先在台账登记 `进行中` 再动文件；工具可加一个
   `--claim <file>` / `--status` 辅助命令，或用锁文件目录降低竞态。
2. `--recommend` 已实现（迭代10 文档曾引用但当时不存在，现已补上）；建议让
   `--recommend` 跳过“已在台账 进行中/验收通过”的文件。
3. 明确单文件验收三连命令为唯一标准：
   `analyze_docstrings.py <file>` → `pytest <file>` → `git diff -- <file>`。

### 发现 C 🟠 — `__init__.py` 包级 docstring 收尾（21 个）

### 目标

1. 补全 `tests/functional/strategies` 全部 `test_*.py` 的结构化 module docstring
   （`Data Used:` / `Strategy Principle:` / `Strategy Logic:`）与类/方法/函数

4. 补齐 `backtrader/` 源码 20 文件 / 195 处缺失 docstring。
5. 工具与文档口径对齐，并恢复/重建迭代10 进度台账。

### 全局硬约束（沿用迭代10 + 质量计划）

```text
[强制] 只增注释/docstring/comment，不改任何可执行逻辑、断言、期望值、参数、
       导入、变量名、数据路径（含 source_ea/source_spec 值）、测试数据。
[强制] 新增文档一律英文，避免被工具计入 Chinese comments。
[强制] 每文件单独验收：analyze_docstrings → pytest → git diff（只见注释变更）。
[强制] 每次只领一个文件，先登记台账 进行中，验收通过后再领下一个。
[强制] 禁止批量脚本改写、禁止后台任务、禁止子代理。
[强制] 不得为通过 pytest 而跳过/放宽断言。
[强制] 补源码 docstring 不得让 mypy 错误越过 gate(≤160) 或引入 ruff 告警。
```
