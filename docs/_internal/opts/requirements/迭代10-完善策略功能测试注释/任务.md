## 1. 迭代目标

本迭代的目标是完善 `tests/functional/strategies` 下功能测试脚本的注释与 Google Style docstring，使每个策略测试文件都能清楚说明：

1. **使用的数据**：数据文件、标的、周期、时间范围、重采样或多数据源关系。
2. **策略原理**：策略依赖的市场假设、指标含义、信号来源和风险控制思路。
3. **策略逻辑**：初始化、入场、出场、订单处理、指标统计和断言校验流程。

最终每个文件必须同时满足：

1. `python scripts/analyze_docstrings.py <目标文件>` 显示 `Google style docstring issues: 0`。
2. `python scripts/analyze_docstrings.py <目标文件>` 显示 `Chinese comments: 0`。
3. `pytest <目标文件>` 通过。
4. `git diff -- <目标文件>` 只能看到新增或补充注释/docstring，不允许出现业务代码、断言、参数、数据路径或执行逻辑变更。
5. 每个 `test_*.py` 策略脚本的文件头部 module docstring 必须重新优化，并包含 `Data Used:`、`Strategy Principle:`、`Strategy Logic:` 三个章节。

## 2. 强制约束

```text
[强制] 只允许增加或补充注释、模块 docstring、类 docstring、函数/方法 docstring。
[强制] 不允许修改任何可执行逻辑、断言、期望值、参数、导入、变量名、数据路径、测试数据或格式化无关代码。
[强制] 不允许为了通过 pytest 而跳过测试、放宽断言或修改 expected 结果。
[强制] 新增测试脚本文档必须使用英文，避免被工具计入 Chinese comments。
[强制] 所有策略脚本头部 module docstring 都必须重写或扩展为结构化说明，不能只补函数级 docstring。
[强制] 每次只领取一个文件，单文件验收通过后再领取下一个文件。
```

说明：很多策略测试文件由 `scripts/inline_regression_tests.py` 生成，文件头部可能包含 `do not edit manually`。本迭代只允许补充说明性 docstring/comment，必须保留原有生成说明，不得改动生成出的测试逻辑。

## 3. 任务领取规则

### 3.1 领取入口

领取任务前先运行目录扫描，确认当前仍未完成的文件清单：

```bash
python scripts/analyze_docstrings.py -d tests/functional/strategies
```

从 `FILES NEEDING OPTIMIZATION` 中选择一个 `test_*.py` 文件作为本轮任务。`__init__.py` 文件放到最后统一扫尾，不优先领取。

### 3.2 并行协作规则

多个 AI 模型并行处理时，按以下规则避免冲突：

1. 每个模型一次只领取一个文件。
2. 领取后在下方"领取台账"新增一行，状态设为 `进行中`。
3. 同一文件已有 `进行中` 或 `验收通过` 记录时，其他模型不得重复领取。
4. 单文件验收通过后，将状态改为 `验收通过`，再领取下一个文件。
5. 如果文件在修改前 `pytest <目标文件>` 已失败，状态标为 `阻塞`，记录失败原因，不修改该文件，重新领取其他文件。

## 4. 单文件执行流程

### Step 1：领取前基线检查

```bash
python scripts/analyze_docstrings.py <目标文件>
pytest <目标文件>
```

### Step 2：重新优化模块头部说明

所有 `test_*.py` 策略脚本都必须重新优化文件顶部 module docstring，包含以下三个章节：

```text
Data Used:
    数据文件、标的、时间周期、时间范围、重采样规则、多数据源关系。

Strategy Principle:
    策略使用的市场假设、指标原理、信号触发逻辑、风险控制思想。

Strategy Logic:
    初始化、数据加载、入场、出场、订单通知、指标提取、测试断言流程。
```

### Step 3：补充类、函数、方法 docstring

按 `scripts/analyze_docstrings.py <目标文件>` 报告逐项补齐 Google Style docstring。

### Step 4：本地验收

```bash
python scripts/analyze_docstrings.py <目标文件>
pytest <目标文件>
git diff -- <目标文件>
```

验收输出必须满足：`Google style docstring issues: 0`、`Chinese comments: 0`、`✅ File is fully documented!`、`pytest` 通过、diff 中只出现 docstring/comment 变更。
